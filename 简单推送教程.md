# 简单推送教程（固定 deploy-dark）

本教程只做两件事：

1. 推送到 GitHub 的 `deploy-dark` 分支  
2. 清理 jsDelivr CDN 缓存

适合小白，按顺序一步一步执行即可。

---

## 第 1 步：打开 PowerShell

按 `Win + R`，输入 `powershell`，回车。

---

## 第 2 步：进入项目根目录

复制并执行：

```powershell
cd "f:\SillyTavern\tavern_helper_template-main"
```

---

## 第 3 步：构建 Dark

复制并执行：

```powershell
cd "f:\SillyTavern\tavern_helper_template-main\src\Dark"
pnpm build
```

看到 `built` 或 `✓ built in ...` 就是成功。

---

## 第 4 步：回到根目录，准备 Git

复制并执行：

```powershell
cd "f:\SillyTavern\tavern_helper_template-main"
git status
```

---

## 第 5 步：切到 deploy-dark 分支

复制并执行：

```powershell
git switch deploy-dark
```

如果报错说没有这个分支，就执行：

```powershell
git switch -c deploy-dark
```

---

## 第 6 步：提交改动

复制并执行：

```powershell
git add .
git commit -m "更新 Dark 界面"
```

如果提示 `nothing to commit, working tree clean`，说明这次没有新改动可提交。

---

## 第 7 步：推送到 GitHub 的 deploy-dark

复制并执行：

```powershell
git push -u origin deploy-dark
```

如果你已经绑定过上游分支，后续可以直接：

```powershell
git push
```

---

## 第 8 步：清理 CDN 缓存（非常重要）

把下面 3 个链接分别打开一次（浏览器地址栏直接打开）：

1. <https://purge.jsdelivr.net/gh/hjf619912-ai/dark-status-bar@deploy-dark/dist/Dark/index.html>  
2. <https://purge.jsdelivr.net/gh/hjf619912-ai/dark-status-bar@deploy-dark/dist/Dark/assets/index.css>  
3. <https://purge.jsdelivr.net/gh/hjf619912-ai/dark-status-bar@deploy-dark/dist/Dark/assets/index.js>

看到 `Purged` / `Done` 类似字样即可。

---

## 第 9 步：验证是否更新成功

打开这个链接检查最新页面：

<https://cdn.jsdelivr.net/gh/hjf619912-ai/dark-status-bar@deploy-dark/dist/Dark/index.html?t=123>

说明：

- 能看到新内容：说明 CDN 已更新成功  
- 还是旧内容：等 1~3 分钟再试，或重新执行第 8 步

---

## 常见报错（只保留最常见）

### 1) `fatal: 'origin' does not appear to be a git repository`

先绑定远程仓库（只需一次）：

```powershell
git remote add origin https://github.com/hjf619912-ai/dark-status-bar.git
```

然后再执行：

```powershell
git push -u origin deploy-dark
```

### 2) `pnpm 不是内部或外部命令`

先安装 pnpm（只需一次）：

```powershell
npm install -g pnpm
```

然后回到第 3 步重新执行。
