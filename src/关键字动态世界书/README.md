# 关键字动态世界书（酒馆助手脚本）

## 小白指定：以后要自己加关键字，改哪里？

**只改一个文件、一处配置：**

1. 用 Cursor / VS Code 打开：`src/关键字动态世界书/index.ts`
2. 搜索 **`RULES`**（文件里有一块注释写着 **「小白只改这里」**）
3. 在 `const RULES = [ ... ]` 的方括号里**加一行**（注意每条后面要有英文逗号 `,`）

**示例：已有蒂薇儿，再加了角色「克洛丝」**

```typescript
const RULES: KeywordWorldbookRule[] = [
  { keywords: ['蒂薇儿'], entryNameIncludes: '蒂薇儿' },
  { keywords: ['克洛丝', 'Clos'], entryNameIncludes: '克洛丝' },
];
```

然后在项目根目录执行 **`pnpm build`**，打开 **`dist/关键字动态世界书/index.js`**，全选复制，去酒馆助手脚本里**覆盖**保存。

**别忘了：** 世界书里要有一条（或多条）**条目名称**里包含你写的 `entryNameIncludes`（例如名称里要有「克洛丝」），脚本才会动到那条。

---

在**当前角色绑定的主世界书**里，根据**最近若干条聊天**是否出现配置的关键字，自动切换条目的激活策略：

- **出现关键字**：匹配到的条目改为 **`constant`（蓝灯）** → 只要条目本身是「启用」的，就会稳定进入上下文（仍受概率、递归等影响，与酒馆行为一致）。
- **未出现**：同一批条目改回 **`selective`（绿灯）** → 按条目自己的关键字扫描，与「踏月寻仙」范例里「关回绿灯」一致。

## 与范例的关系

你给的 `踏月寻仙-动态世界书管理` 也是用 `getCharWorldbookNames` + `updateWorldbookWith` 改 `strategy.type`。本脚本去掉 MVU、地图等逻辑，只保留「**词出现 → 条目蓝灯**」这一件事，方便改规则。

## 使用步骤

1. **`pnpm build`** 后，把 `dist/关键字动态世界书/index.js` **全选复制**到酒馆助手 → 新建脚本（或网络 `import` 该文件 URL）。
2. 在角色卡里绑定**主世界书**（脚本只改主世界书 `primary`）。
3. 在世界书里建条目，**条目名称**里要包含你在代码里写的 `entryNameIncludes` 字符串。  
   - 例如规则里写 `entryNameIncludes: '蒂薇儿'`，条目名可以是：`蒂薇儿`、`[角色] 蒂薇儿 - 好感期` 等，只要**包含** `蒂薇儿` 即可。
4. 编辑 `src/关键字动态世界书/index.ts` 里的 **`RULES`**，再重新 build：

```typescript
const RULES: KeywordWorldbookRule[] = [
  { keywords: ['蒂薇儿', '蒂薇'], entryNameIncludes: '蒂薇儿' },
];
```

- **`keywords`**：在近期消息里**任一字串出现**即视为命中（子串匹配）。
- **`entryNameIncludes`**：世界书**条目名称**需包含该子串才会被本脚本改蓝/绿灯。

## 可调参数

| 常量 | 作用 |
|------|------|
| `SCAN_LAST_MESSAGES` | 扫描最近多少条楼层 |
| `DEBOUNCE_MS` | 防抖，避免事件连发时反复写世界书 |
| `DEBUG` | `true` 时在控制台输出 `[关键字世界书]` 日志 |

## 事件

在 **用户发送**、**收到 AI 回复**、**换角色**、**换聊天** 后会防抖执行一次同步。若你希望与范例一样在「仅发送时开、回复后关」，可自行改成只监听 `MESSAGE_SENT` / `MESSAGE_RECEIVED`（当前是两边都会同步到「与当前聊天记录一致」的状态）。

## 若你想换的是「整本世界书」而不是条目灯

那是 **`rebindCharWorldbooks`**（换角色绑定哪几本世界书），和「同一张世界书里某些条目蓝/绿」不同。需要的话可以再说，我可以另写一个小脚本模板。
