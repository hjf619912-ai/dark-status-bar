# 推送根目录（Dark 状态栏）

目的：把 `src/Dark` 构建出来的结果，发布到 Git 仓库 **根目录**，最终仓库只需要保留：
1. `index.html`
2. `assets/`（里面是 `index.css`、`index.js`、字体/图片等）

适用场景：你希望酒馆正则直接从 `@master/` 加载，而不是从 `@master/dist/Dark/` 加载。

---

## 0. 前置准备

1. 确保机器上已装好 `git`、`pnpm`
2. 确保你有推送权限（能推 `master`）
3. 建议先备份一个分支（避免操作失误）

```powershell
cd D:\服务器上资源\tavern_helper_template-main
git branch backup-before-dark-root
```

---

## 1. 构建 Dark（生成 dist）

```powershell
cd D:\服务器上资源\tavern_helper_template-main\src\Dark
pnpm install
pnpm build
```

构建完成后，你会得到：
- `D:\服务器上资源\tavern_helper_template-main\dist\Dark\index.html`
- `D:\服务器上资源\tavern_helper_template-main\dist\Dark\assets\*`

---

## 2. 把 dist/Dark 的内容复制到仓库根目录

```powershell
cd D:\服务器上资源\tavern_helper_template-main

# 先删旧的根目录发布文件（避免残留）
Remove-Item -Recurse -Force .\assets -ErrorAction SilentlyContinue
Remove-Item -Force .\index.html -ErrorAction SilentlyContinue

# 复制到根目录
New-Item -ItemType Directory -Force -Path .\assets | Out-Null
Copy-Item -Path .\dist\Dark\assets\* -Destination .\assets -Recurse -Force
Copy-Item -Path .\dist\Dark\index.html -Destination .\index.html -Force

# 验证：确认根目录里有 index.html 和 assets/index.js
dir .\index.html
dir .\assets\index.js
dir .\assets\index.css
```

---

## 3. 用 Git 只保留根目录的 index.html + assets/

1. 回到仓库根目录
2. 把“缓存索引”里其它文件全部取消跟踪
3. 只加入你要发布的两个部分

```powershell
cd D:\服务器上资源\tavern_helper_template-main

git rm -r --cached .
git add index.html assets/
git status
```

`git status` 里你应该看到待提交的只有：
- `index.html`
- `assets/*`

---

## 4. 提交并推送到 GitHub（覆盖 master）

```powershell
git commit -m "release: publish Dark to repo root"
git push origin master --force-with-lease
```

如果 Git 提示你“需要先 pull / non-fast-forward”，告诉我报错原文即可；但你现在的目标就是覆盖 master（等同于“第一次推送”）。

---

## 5. （可选但强烈建议）更新正则与清 CDN

发布到根目录后，你的酒馆正则加载地址应当改成：
`https://cdn.jsdelivr.net/gh/hjf619912-ai/dark-status-bar@master`

然后用 jsDelivr 的 purge 清缓存（打开三个链接各一次）：
- `https://purge.jsdelivr.net/gh/hjf619912-ai/dark-status-bar@master/index.html`
- `https://purge.jsdelivr.net/gh/hjf619912-ai/dark-status-bar@master/assets/index.css`
- `https://purge.jsdelivr.net/gh/hjf619912-ai/dark-status-bar@master/assets/index.js`

---

## 常见问题

### 1）我推送后手机还是旧版
通常是 CDN 或浏览器缓存导致。把上面 purge 做一遍，再在手机上完全关闭页面重开。

### 2）`assets/` 很多但不进仓库
你需要确保步骤 3 的 `git add index.html assets/` 成功，且 `git status` 里只剩这些待提交文件。
