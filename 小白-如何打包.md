# 小白：改完代码后怎么打包（Build）

不用道歉，照着做就行。

---

## 你要进哪个文件夹？

进**本项目的根目录**——就是里面能看到 **`package.json`** 的那一层。

你电脑上的路径一般是：

```text
f:\SillyTavern\tavern_helper_template-main
```

**不要**进到 `src` 里面再 build；**不要**进到 `dist` 里面。就在**和 `package.json` 同级**的这一层。

---

## 怎么 Build？（Windows）

1. 用资源管理器打开上面的文件夹 `tavern_helper_template-main`。
2. 在文件夹空白处 **Shift + 右键** → 选 **「在此处打开 PowerShell 窗口」** 或 **「在终端中打开」**。
3. 输入下面这一行，按回车：

```powershell
pnpm build
```

4. 等它跑完，没有大红字报错就行。

**第一次**在这台电脑上如果提示没有 `pnpm`，需要先装 Node.js，再在终端执行：

```powershell
npm install -g pnpm
```

---

## Build 完之后，去复制哪个文件进酒馆助手？

| 你改的是哪个脚本 | 复制这个文件（全选复制内容） |
|------------------|------------------------------|
| 关键字动态世界书 | `dist\关键字动态世界书\index.js` |
| Dark 状态栏加载脚本 | `dist\Dark状态栏加载脚本\index.js` |

路径仍是：在**项目根目录**下的 `dist` 文件夹里。

---

## 记住两句话

1. **改代码** → 打开 `src\某某\index.ts`（不是 `.js`）。
2. **给酒馆用** → 必须在根目录执行 **`pnpm build`**，再用 **`dist\...` 里的 `index.js`**。

---

## 想自动监视、改完就编？

在项目根目录终端执行（开着别关）：

```powershell
pnpm watch
```

保存 `src` 里的文件后会自动重新打包（需配合酒馆助手「实时监听」使用）。
