# Dark 状态栏 · 推送到 GitHub 超详细小白指南

**目标**：把本地改好的 Dark 状态栏，上传到 GitHub，让酒馆能通过 jsDelivr 加载显示。

**适用**：从未用过命令行、Git、GitHub 的人。照着做就行。

---

## 零、你需要提前知道的事

### 0.1 什么叫「复制命令」？

1. 用鼠标**选中**命令（拖拽选中整行）
2. 按 **Ctrl + C** 复制
3. 在 PowerShell 窗口里**点一下**（确保光标在里面）
4. **右键点击** → 会自动粘贴（PowerShell 里 Ctrl+V 有时无效，右键一定可以）

### 0.2 什么叫「回车」？

按键盘上的 **Enter** 键。每复制一条命令、粘贴后，都要按一次回车执行。

### 0.3 你的项目路径

本指南默认你的项目在：

```
f:\SillyTavern\tavern_helper_template-main
```

如果你的项目在别的盘或别的文件夹，请把下面所有命令里的这个路径改成你自己的。

## 一、首次使用前的准备（只需做一次）

### 1.1 安装 Git

1. 打开浏览器，访问：**https://git-scm.com/download/win**
2. 页面会自动推荐 **64-bit Git for Windows Setup**，点它下载
3. 下载完成后，双击安装包
4. 安装过程中，**所有选项保持默认**，一路点 **Next**
5. 最后点 **Finish** 完成
6. **关掉所有已经打开的 PowerShell 或 CMD 窗口**
7. 重新打开一个新的 PowerShell（见下一节）

---

### 1.2 打开 PowerShell

**方法一（推荐）**：

1. 按键盘 **Win + R**（Win 键在 Ctrl 和 Alt 旁边）
2. 弹出小窗口里输入：`powershell`
3. 按回车

**方法二**：

1. 点左下角「开始」菜单
2. 搜索 **PowerShell**
3. 点击打开「Windows PowerShell」

打开后，会看到一个蓝色或黑色的窗口，最后一行类似：

```
PS C:\Users\你的用户名>
```

---

### 1.3 配置 Git 用户名和邮箱（只需做一次）

把下面两行**分别**复制到 PowerShell，每行粘贴后按回车：

```powershell
git config --global user.name "hjf619912-ai"
```

```powershell
git config --global user.email "13530999+evidence-of-dawn@user.noreply.gitee.com"
```

> 如果你想用自己 GitHub 绑定的邮箱，把第二行的邮箱改成你的。

**如果成功**：不会有任何提示，直接回到 `PS ...>`，说明配置好了。

---

### 1.4 添加 GitHub 远程仓库（只需做一次）

依次复制下面两行，每行粘贴后按回车：

```powershell
cd "f:\SillyTavern\tavern_helper_template-main\deploy-dark"
```

```powershell
git remote add github https://github.com/hjf619912-ai/dark-status-bar.git
```

**如果出现**：`error: remote github already exists`

说明你之前已经添加过了，**忽略即可**，继续后面的步骤。

## 二、日常更新推送（每次改完 Dark 代码后都要做）

下面是一整套流程，**按顺序**一条一条执行，不要跳步。

---

### 第 1 步：进入 Dark 项目目录

复制这条，粘贴到 PowerShell，回车：

```powershell
cd "f:\SillyTavern\tavern_helper_template-main\src\Dark"
```

**如果成功**：下一行会变成类似 `PS F:\SillyTavern\tavern_helper_template-main\src\Dark>`

---

### 第 2 步：执行构建

复制这条，粘贴，回车：

```powershell
pnpm build
```

**如果成功**：最后会看到类似：

```
✓ built in 1.5s
```

**如果失败**，出现 `pnpm 不是内部或外部命令`：

先执行（安装 pnpm）：

```powershell
npm install -g pnpm
```

然后再执行一次 `pnpm build`。

---

### 第 3 步：回到项目根目录

```powershell
cd "f:\SillyTavern\tavern_helper_template-main"
```

---

### 第 4 步：把构建好的文件复制到 deploy-dark

```powershell
Copy-Item -Path "dist\Dark\*" -Destination "deploy-dark\" -Recurse -Force
```

**如果成功**：没有任何提示，直接回到 `PS ...>`

---

### 第 5 步：进入 deploy-dark 目录

```powershell
cd deploy-dark
```

---

### 第 6 步：把修改加入暂存区

```powershell
git add .
```

**如果成功**：没有任何提示。

---

### 第 7 步：提交（写一句说明）

```powershell
git commit -m "更新"
```

你可以把 `"更新"` 改成别的，比如 `"手机端设置优化"`，只要在英文引号里就行。

**如果成功**：会看到类似：

```
[master xxxxx] 更新
 X files changed, ...
```

**如果出现**：`nothing to commit, working tree clean`

说明没有新改动。请确认第 1～4 步都做过了，再重试。

---

### 第 8 步：推送到 GitHub

```powershell
git push github master
```

**如果成功**：会看到类似：

```
Enumerating objects: ...
...
To https://github.com/hjf619912-ai/dark-status-bar.git
   xxxxx..yyyyy  master -> master
```

**如果出现登录窗口**：按提示用 GitHub 账号登录，或输入 **Personal Access Token** 作为密码（不是登录密码）。

**如果出现**：`Failed to connect to github.com port 443`

说明连不上 GitHub，见「常见问题」里的解决方法。

---

### 第 9 步（可选）：同时推送到 Gitee

如果你也用到 Gitee，再执行：

```powershell
git push origin master
```

## 三、一整段复制版（适合已经熟悉流程的人）

**适合**：已经做过一次完整流程、知道每一步在干嘛的人。

**用法**：复制下面**整段**，粘贴到 PowerShell，按一次回车，会依次执行所有命令。

```powershell
cd "f:\SillyTavern\tavern_helper_template-main\src\Dark"; pnpm build; cd "f:\SillyTavern\tavern_helper_template-main"; Copy-Item -Path "dist\Dark\*" -Destination "deploy-dark\" -Recurse -Force; cd deploy-dark; git add .; git commit -m "更新"; git push github master
```

如果要同时推 Gitee，再复制执行：

```powershell
git push origin master
```

## 四、酒馆里用的加载脚本（直接复制）

推送成功后，在酒馆的「正则替换」或「自定义 HTML」里，把下面的内容**整段复制**进去，替换掉原来的 `<body>...</body>`：

```html
<body>
<script>
$(() => {
  const baseUrl = 'https://cdn.jsdelivr.net/gh/hjf619912-ai/dark-status-bar@master';
  const pageUrl = baseUrl + '/index.html';
  $.get(pageUrl, (htmlText) => {
    const doc = new DOMParser().parseFromString(htmlText, 'text/html');
    const root = doc.querySelector('#root');
    const rootHtml = root ? root.outerHTML : '<div id="root"></div>';
    const wrap = '<div class="dark-embed-wrap" style="width:100%;min-width:100%;max-width:100vw;overflow-x:hidden;box-sizing:border-box;">' + rootHtml + '</div>';
    $('body').html(wrap);
    $('body').css({ overflowX: 'hidden', width: '100%', maxWidth: '100vw' });
    $('html').css('overflowX', 'hidden');
    $('head').find('link[data-dark-css="1"]').remove();
    $('head').append('<link rel="stylesheet" data-dark-css="1" href="' + baseUrl + '/assets/index.css">');
    const old = document.getElementById('dark-remote-module');
    if (old) old.remove();
    const s = document.createElement('script');
    s.id = 'dark-remote-module';
    s.type = 'module';
    s.src = baseUrl + '/assets/index.js';
    s.crossOrigin = 'anonymous';
    document.body.appendChild(s);
  });
});
</script>
</body>
```

**注意**：复制时从 `<body>` 到 `</body>` 要完整，不要漏掉开头或结尾。

推送后约 1～2 分钟生效。想立刻看到效果，可打开 **https://www.jsdelivr.com/tools/purge-cache** 清除缓存。

## 五、常见问题

### Q1：`pnpm` 不是内部或外部命令

**原因**：没装 pnpm。

**解决**：在 PowerShell 执行：

```powershell
npm install -g pnpm
```

然后再执行 `pnpm build`。

---

### Q2：`Failed to connect to github.com port 443`

**原因**：国内网络经常连不上 GitHub。

**解决一：用 ghproxy 代理**

依次执行：

```powershell
git remote set-url github https://ghproxy.com/https://github.com/hjf619912-ai/dark-status-bar.git
```

```powershell
git push github master
```

推送成功后可改回直连（可选）：

```powershell
git remote set-url github https://github.com/hjf619912-ai/dark-status-bar.git
```

**解决二**：开 VPN 后再执行 `git push github master`。

---

### Q3：推送时弹出窗口要输入用户名和密码

- **用户名**：填你的 GitHub 用户名（例如 `hjf619912-ai`）
- **密码**：填 **Personal Access Token**，**不是** GitHub 登录密码

**怎么创建 Token**：

1. 打开 https://github.com/settings/tokens
2. 点 **Generate new token** → **Generate new token (classic)**
3. **Note** 随便填，如 `dark`
4. 勾选 **repo**
5. 点 **Generate token**
6. 复制生成的令牌（只显示一次），粘贴到「密码」框

---

### Q4：`remote github already exists` / `remote origin already exists`

**原因**：已经添加过这个远程了。

**解决**：不用管，直接执行后面的 `git push github master` 或 `git push origin master` 即可。

---

### Q5：`Author identity unknown` / `Please tell me who you are`

**原因**：没配置 Git 用户名和邮箱。

**解决**：执行 1.3 里的两行配置命令，然后再 `git commit`。

---

### Q6：`nothing to commit, working tree clean`

**原因**：没有新的文件改动需要提交。

**解决**：确认第 1～4 步（构建、复制）都做过了。如果 `dist\Dark` 里确实有新内容，再执行一次第 4 步的复制命令，然后重新 `git add .` 和 `git commit`。

---

### Q7：`src refspec main does not match any`

**原因**：分支名不对，你本地是 `master` 不是 `main`。

**解决**：用 `master`：

```powershell
git push github master
```

不要用 `main`。

---

### Q8：复制粘贴后命令没反应 / 报错

- 确认是**整行**复制，没有多出空格或换行
- 确认路径 `f:\SillyTavern\tavern_helper_template-main` 和你的实际路径一致
- 如果路径含中文或空格，整条命令要放在英文双引号里
